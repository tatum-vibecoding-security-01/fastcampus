# 기획서 — 카카오톡 대화 기반 연애 심리 상담 서비스 (가칭: **썸닥터** / SomeDoctor)

> 카카오톡 대화 내보내기 텍스트를 분석해 관계 온도를 진단하고, 데이터 기반 분석가 톤으로 맞춤 조언과 답장 코칭을 대화형으로 제공하는 반응형 웹 서비스

- 문서 버전: v0.1 (초안)
- 작성일: 2026-07-12
- 문서 수준: 실무급 상세 (개발 착수용)

---

## 1. 개요 (Overview)

### 1.1 한 줄 정의
썸/탐색전 단계의 사용자가 카카오톡 대화 내용을 업로드하면, **관계 온도를 진단**하고 **다음에 어떻게 답장·행동해야 할지** 데이터 기반으로 코칭해주는 대화형 웹 서비스.

### 1.2 배경 & 문제 정의
- 썸·초기 관계 단계의 사람들은 "이 사람이 나에게 관심이 있나?", "이 답장 이렇게 보내도 될까?"에 대한 확신이 없어 **혼자 대화를 수백 번 곱씹는다.**
- 친구에게 대화를 캡처해 보여주며 조언을 구하지만, 친구는 편향적이고 즉시 응답하지 못하며 반복 질문이 부담스럽다.
- 기존 사주/타로 앱은 근거가 빈약하고, 심리 상담은 비싸고 문턱이 높다.

### 1.3 해결책 (Value Proposition)
- **객관성**: 감정이 아닌 대화 데이터(응답 속도, 빈도, 대화 주도권, 감정 표현 등)에 근거한 진단.
- **즉시성**: 24시간, 몇 초 만에, 반복 질문에도 지치지 않는 상담사.
- **실행 가능성**: "관심 있어요/없어요" 수준이 아니라 **다음 답장 문구까지** 구체적으로 코칭.

### 1.4 핵심 차별점
| 대안 | 한계 | 본 서비스의 강점 |
|---|---|---|
| 친구 조언 | 편향·비일관·눈치보임 | 중립적·언제나·무제한 |
| 사주/타로 | 근거 없음 | 실제 대화 데이터 기반 |
| 심리 상담 | 비쌈·예약 필요 | 무료 진단 + 저렴한 코칭 |

## 2. 타겟 사용자 (Target Users)

### 2.1 주요 타겟
**썸·탐색전 단계**의 20~30대. 아직 연인은 아니지만 특정 상대와의 관계 발전 가능성을 궁금해하고, 매 답장에 신중을 기하는 사람.

### 2.2 페르소나
**"밤마다 대화를 곱씹는 지현 (28세, 직장인)"**
- 소개팅으로 만난 상대와 2주째 카톡 중. 하루 종일 답장을 기다리고, 상대의 짧은 답장에 의미를 부여한다.
- "내가 더 좋아하는 것 같은데… 이 사람도 나한테 관심 있는 걸까?"
- 친구에게 물어보기엔 이미 너무 많이 물어봤다. 객관적인 제3자의 판단이 필요하다.

### 2.3 사용자의 핵심 니즈 (Jobs To Be Done)
1. "이 사람이 나한테 관심 있는지 **확인**하고 싶다." → 진단
2. "이 상황에서 **어떻게 답장**해야 할지 알고 싶다." → 코칭
3. "내가 **너무 들이대거나 애매하게 굴고 있진 않은지** 알고 싶다." → 나의 패턴 피드백

## 3. 핵심 기능 (MVP Scope)

### 3.1 MVP 기능 우선순위
| 우선순위 | 기능 | 설명 | 무료/유료 |
|---|---|---|---|
| P0 | 대화 업로드 & 파싱 | 카톡 내보내기 텍스트 업로드 → 화자/시간/메시지 구조화 | 무료 |
| P0 | 관계 온도 진단 | 대화 지표 기반 온도(0~100°) + 핵심 지표 카드 | 무료 (요약) / 유료 (상세) |
| P0 | 대화형 채팅 상담 | 진단 결과를 바탕으로 사용자와 채팅하며 조언·답장 코칭 | 무료 (제한 횟수) / 유료 (무제한) |
| P1 | 답장 코칭 | "이렇게 답장해보세요" 구체 문구 3안 제시 | 유료 |
| P1 | 나의 대화 패턴 피드백 | 사용자 본인의 습관(과한 답장/집착 신호 등) 리포트 | 유료 |
| P2 | 감정 흐름 타임라인 | 시간에 따른 호감/권태 신호 시각화 | 유료 |

### 3.2 기능 상세

#### (P0) 대화 업로드 & 파싱
- **입력**: 카카오톡 "대화 내보내기"로 생성된 `.txt` 파일 (또는 텍스트 붙여넣기).
- **파싱 요구사항**:
  - PC/모바일 내보내기 포맷 모두 지원 (날짜 구분선, `[이름] [오전 11:23] 메시지` 형식 등).
  - 화자 식별 → 사용자에게 "당신은 누구인가요?" 선택 요청 (본인/상대 지정).
  - 시스템 메시지(입장/나감, 사진, 이모티콘) 정규화 처리.
- **제약**: 1:1 대화만 지원(MVP). 단체방 제외.
- **엣지 케이스**: 이름 중복, 개명, 저장된 이름과 표시 이름 불일치 → 화자 매핑 UI 제공.

#### (P0) 관계 온도 진단
- 대화를 분석해 **관계 온도(0~100°)** 와 근거 지표를 산출.
- **정량 지표 (규칙 기반 계산)**:
  - 응답 속도(양측 평균/중앙값, 비대칭성)
  - 대화 주도권(누가 먼저 말 거는가, 먼저 카톡 비율)
  - 메시지 길이·빈도 밸런스
  - 질문 비율(상대가 나에게 질문하는가 = 관심 신호)
  - 대화 지속 시간, 끊김 패턴(누가 먼저 끊는가)
  - 이모티콘/감정 표현 빈도
- **정성 지표 (LLM 분석)**:
  - 감정 톤, 호감/거리두기 신호, 밀당 패턴, 대화 온도 변화
- **결과 출력**: 온도 + "핵심 신호 3가지" + 한 줄 요약. 상세 근거는 유료.

#### (P0) 대화형 채팅 상담 (핵심 UX)
- 진단 결과 화면에서 자연스럽게 채팅 상담으로 진입.
- **페르소나: 데이터 기반 분석가** (아래 4.3 참고).
- 사용자가 "왜 온도가 낮게 나왔어?", "이 답장 어때?", "이럴 땐 어떻게 해?" 등 자유 질문.
- 상담사는 **업로드된 대화 데이터를 근거로** 답변 (예: "상대방 평균 답장 시간이 3시간인데, 이건 관심 부족보다는 라이프스타일일 가능성이 높습니다. 실제로 밤 10시 이후 응답 속도는 12분으로 빨라집니다.")
- 무료: 하루 N회 제한 / 유료: 무제한.

#### (P1) 답장 코칭
- 사용자가 상대의 최근 메시지를 입력하거나 대화에서 선택 → **답장 3안** 제시.
- 각 안마다 톤(안전형/적극형/위트형)과 예상 효과를 데이터 근거로 설명.

#### (P1) 나의 대화 패턴 피드백
- 사용자 본인의 습관 진단: 과한 답장 빈도, 늦은 밤 연속 메시지, 질문 공세, 감정 과잉 표현 등.
- "개선 제안"을 데이터 근거와 함께 제공.

## 4. 상담 페르소나 & 톤 (Persona Design)

### 4.1 페르소나: "데이터 기반 분석가"
- 감정적 위로보다 **객관적 사실과 근거**로 신뢰를 준다.
- 냉정하되 무례하지 않다. 판단을 강요하지 않고 데이터를 제시한 뒤 해석을 돕는다.
- 항상 **수치·패턴·구체적 대화 예시**를 인용한다.

### 4.2 톤 가이드
| 해야 할 것 | 하지 말아야 할 것 |
|---|---|
| "응답 속도 데이터를 보면…" | "제 느낌엔…" (근거 없는 직관) |
| "확정할 수 없지만, ~일 가능성이 높습니다" | "이 사람은 당신을 사랑해요" (단정) |
| 사용자의 선택을 존중 | 특정 행동 강요 |
| 희망/불안 균형 있게 | 과도한 낙관 또는 비관 |

### 4.3 안전 가이드라인 (필수)
- **의료/정신건강 위기 감지**: 자해·극단적 표현 시 상담 중단 및 전문기관(예: 자살예방상담 109) 안내.
- **폭력·스토킹 조장 금지**: 상대의 거절 신호가 명확할 때 집착·추적을 조언하지 않는다.
- **단정 금지**: 관계 판단은 확률·경향으로 표현. 면책 문구 상시 노출.
- **미성년자·비동의 데이터**: 서비스 약관에 명시.

## 5. 사용자 플로우 (User Flow)

```
[랜딩] 
  → [대화 업로드 / 붙여넣기]
  → [화자 지정: "당신은 누구?"]
  → [분석 중… (수 초, 로딩 애니메이션)]
  → [무료 진단 결과: 온도 + 핵심 신호 3 + 한 줄 요약]
  → 분기
      ├─ [대화형 채팅 상담 시작] (무료 N회)
      │     → 질문/답장 코칭 반복
      │     → 무료 한도 도달 → [유료 전환 유도]
      └─ [상세 리포트 보기] → [유료 전환 유도(Paywall)]
  → [분석 종료 시 데이터 즉시 파기 안내]
```

### 5.1 핵심 화면 목록
1. **랜딩 페이지**: 가치 제안, "무료로 진단하기" CTA, 개인정보 파기 정책 강조(신뢰 확보).
2. **업로드 화면**: 파일 업로드 / 텍스트 붙여넣기, 내보내기 방법 가이드(스크린샷 포함).
3. **화자 지정 화면**: 파싱된 화자 목록에서 본인 선택.
4. **로딩 화면**: 분석 진행 상태 + "데이터는 저장되지 않습니다" 재확인.
5. **진단 결과 화면**: 온도 게이지, 핵심 신호 카드, 요약. 하단에 채팅 상담 진입 + Paywall.
6. **채팅 상담 화면**: 대화형 UI. 사용자 메시지 입력, 상담사 응답, 답장 코칭 카드.
7. **Paywall/결제 화면**: Freemium 업그레이드.
8. **약관/개인정보 처리방침**.

## 6. 데이터 처리 & 보안 (Data & Privacy) — **최우선 신뢰 요소**

### 6.1 원칙: 분석 후 즉시 파기 (비저장)
- 사용자의 카톡 대화는 **서버 DB에 영구 저장하지 않는다.**
- 처리 흐름:
  1. 클라이언트에서 업로드 → 서버는 메모리(또는 휘발성 세션 스토리지)에서만 처리.
  2. LLM 분석 요청 → 응답 수신.
  3. 세션 종료 또는 분석 완료 시 원문·중간 결과 **즉시 삭제**.
- 채팅 상담 중에는 세션 동안만 대화 컨텍스트 유지(TTL 설정, 예: 30분 무활동 시 파기).

### 6.2 개인정보 최소화
- 실명·전화번호 등은 파싱 단계에서 익명화(화자 A/B 또는 별칭)한 뒤 LLM 전송 검토.
- 로그에 대화 원문을 남기지 않는다(요청/응답 본문 로깅 금지).

### 6.3 LLM 전송 관련 고지
- 분석을 위해 대화 텍스트가 Claude API로 전송됨을 명시적으로 고지·동의받는다.
- API 제공사의 데이터 미학습(Zero-retention) 옵션 적용 여부 확인 및 명시.

### 6.4 신뢰 UX
- 랜딩·업로드·결과 화면에 "저장하지 않습니다 / 분석 후 즉시 삭제" 배지 반복 노출.
- 처리방침을 쉬운 말로 요약 제공.

> ⚠️ **법무 검토 필요 항목**: 개인정보처리방침, 제3자(상대방) 정보 처리의 적법성, 국외 이전(LLM API) 고지, 전자상거래법(결제) 준수.

## 7. 기술 설계 (Technical Design)

### 7.1 기술 스택 (제안)
| 영역 | 선택(제안) | 비고 |
|---|---|---|
| 프론트엔드 | Next.js (React) + TypeScript | 반응형, SSR/SEO(랜딩) |
| 스타일 | Tailwind CSS | 빠른 반응형 구현 |
| 백엔드 | Next.js API Routes 또는 Node(Express/Fastify) | 서버리스 가능 |
| LLM | **Claude API (claude-opus-4-8 / claude-sonnet-5)** | 분석·상담. 비용/품질에 따라 모델 선택 |
| 파싱 | 자체 파서(정규식 기반) | 카톡 포맷별 대응 |
| 세션 | 인메모리 / Redis(TTL) | 비저장 원칙 준수, 세션 만료 자동 파기 |
| 결제 | 토스페이먼츠 / 아임포트(PortOne) | 국내 결제 |
| 인프라 | Vercel 또는 AWS | 초기 비용 최소화 |
| 분석/모니터링 | 개인정보 비수집 방식의 웹 분석 | 대화 원문 미수집 |

### 7.2 시스템 아키텍처 (개념)
```
[브라우저]
   │ (업로드/채팅)
   ▼
[Next.js Frontend] ── [API Layer]
                          │
             ┌────────────┼───────────────┐
             ▼            ▼                ▼
        [KakaoParser] [규칙기반          [Claude API]
        (구조화)      지표 계산 엔진]     (정성 분석/상담)
             │            │                │
             └──── [세션 컨텍스트(TTL, 비영구)] ────┘
                          │
                    (세션 종료 시 파기)
```

### 7.3 분석 엔진: 규칙 기반 + LLM 하이브리드
- **규칙 기반 계산 레이어**(비용 절감 + 일관성): 응답 속도, 빈도, 주도권, 질문 비율 등 정량 지표를 코드로 계산.
- **LLM 레이어**(해석 + 상담): 계산된 지표 + 대화 샘플을 컨텍스트로 넣어 온도 해석, 신호 요약, 대화형 상담 수행.
- 이유: 전체 대화를 매번 LLM에 다 넣으면 비용·토큰 부담 큼. 정량은 코드로, 정성/대화는 LLM으로 분담.

### 7.4 관계 온도 산출 로직 (초안)
- 각 지표를 0~1로 정규화 후 가중 합산 → 0~100° 매핑.
- 예시 가중치(튜닝 대상):
  - 응답 속도 대칭성 20%
  - 대화 주도권 균형 15%
  - 상대의 질문 비율 20%
  - 대화 빈도·지속성 15%
  - 감정/이모티콘 표현 15%
  - LLM 정성 판단(보정) 15%
- ⚠️ 초기 가중치는 가설. **사용자 피드백/실사용 데이터로 보정 로드맵 필요**(단, 원문 비저장 원칙과 충돌하지 않는 익명 통계 방식으로).

### 7.5 LLM 프롬프트 설계 (개요)
- **시스템 프롬프트**: 페르소나(데이터 기반 분석가), 안전 가이드라인, 출력 형식, 단정 금지 규칙.
- **컨텍스트 주입**: 규칙 기반 지표(JSON) + 대표 대화 발췌 + 사용자 질문.
- **출력 스키마화**: 진단 결과는 구조화된 형식(온도, 신호 배열, 요약)으로 강제.
- **가드레일**: 위기 신호 감지 시 정해진 안내 메시지로 분기.

## 8. 수익 모델 (Monetization)

### 8.1 Freemium 구조
| 구분 | 무료 | 유료 (Premium) |
|---|---|---|
| 관계 온도 진단 | 온도 + 핵심 신호 3 + 한 줄 요약 | 전체 지표 상세 리포트 |
| 대화형 채팅 상담 | 하루 N회(예: 3회) | 무제한 |
| 답장 코칭 | 미리보기 1안 | 3안 + 상세 근거 |
| 나의 패턴 피드백 | 잠금 | 제공 |
| 감정 흐름 타임라인 | 잠금 | 제공 |

### 8.2 과금 방식 (검토안)
- **구독**: 월 정액(무제한 상담) — 반복 사용 유도.
- **단건**: 1회 상세 리포트 이용권 — 1회성 사용자 대응.
- MVP는 우선 **단건 + 월 구독** 병행 A/B 검토.

### 8.3 가격 (가설, 시장 검증 필요)
- 단건 상세 리포트: ₩2,900~4,900
- 월 구독: ₩9,900 내외
